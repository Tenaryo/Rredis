1. Do not add any unnecessary comments. If you have to add comments, use English with consistent format.
2. Avoid writing boilerplate or generic, repetitive code.
3. Focus on modern features (e.g., C++17/20/23/26), performance, and scalability.
4. Ensure proper modularization; no single function should exceed 100 lines.
5. Favor compile-time computation when possible (constexpr, templates).
6. Write cache-friendly and data-oriented code where performance matters.
7. Pay attention to architectural scalability.
8. Be critical, able to challenge user's idea if his plan is not optimal.
9. After you fully complete your work, use clang-format to format your code.
10. Keep the main function clean and clear.
11. Don't git commit for the user, user will commit himself.
